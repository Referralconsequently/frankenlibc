# Gentoo Validation User Guide

This guide is for users who want to run Gentoo package validation with FrankenLibC interposition and reproduce the published ecosystem results.

## Quick Start

Run this from the repo root:

```bash
# 1) Build the Gentoo validation image
docker build -f docker/gentoo/Dockerfile.frankenlibc -t frankenlibc/gentoo-frankenlibc:latest .

# 2) Run fast validation (Tier-1 package set)
scripts/gentoo/fast-validate.sh --hardened

# 3) Generate a markdown report from produced artifacts
python3 scripts/gentoo/generate-report.py \
  --franken-version 0.1.0 \
  --gentoo-stage3 2026-02-01 \
  --output docs/gentoo/VALIDATION-REPORT.md

# 4) Lint documentation/link integrity
python3 scripts/gentoo/validate-docs.py --strict
```

## Prerequisites

- Linux host with Docker.
- FrankenLibC repository checked out.
- Python 3.11+ available as `python3`.
- Optional: root/system-level Portage environment if validating outside Docker.

## Docker Setup

Use the Dockerfiles in [`docker/gentoo/`](../../docker/gentoo):

- `Dockerfile.stage3`: base stage3 image bootstrap.
- `Dockerfile.builder`: builder image with package tooling.
- `Dockerfile.frankenlibc`: FrankenLibC-integrated validation image.

Recommended build:

```bash
docker build -f docker/gentoo/Dockerfile.frankenlibc -t frankenlibc/gentoo-frankenlibc:latest .
```

## Configuration

Core configuration files:

- [`configs/gentoo/build-config.toml`](../../configs/gentoo/build-config.toml): runner defaults.
- [`configs/gentoo/portage-bashrc`](../../configs/gentoo/portage-bashrc): hook env template.
- [`configs/gentoo/exclusions.json`](../../configs/gentoo/exclusions.json): package exclusion policy.
- [`configs/gentoo/tier1-mini.txt`](../../configs/gentoo/tier1-mini.txt): fast validation package set.

Most important runtime knobs:

- `FRANKENLIBC_MODE=strict|hardened`
- `FRANKENLIBC_PORTAGE_ENABLE=0|1`
- `FRANKENLIBC_PHASE_ALLOWLIST="src_test pkg_test"`
- `FRANKENLIBC_PACKAGE_BLOCKLIST="sys-libs/glibc ..."`
- `FRANKENLIBC_LIB=/opt/frankenlibc/lib/libfrankenlibc_abi.so`

## Running Builds

## 1) Fast lane (recommended first)

```bash
scripts/gentoo/fast-validate.sh --hardened
```

Artifacts are written under `artifacts/gentoo-builds/fast-validate/<timestamp>/`.

## 2) Full build runner lane

```bash
python3 scripts/gentoo/build-runner.py --config configs/gentoo/build-config.toml
```

Limit to selected atoms:

```bash
python3 scripts/gentoo/build-runner.py \
  --config configs/gentoo/build-config.toml \
  --package sys-apps/coreutils \
  --package net-misc/curl
```

## 3) Baseline vs instrumented test lane

```bash
python3 scripts/gentoo/test-runner.py \
  --package sys-apps/coreutils \
  --frankenlibc-mode hardened \
  --output artifacts/gentoo-tests
```

## Interpreting Results

- Fast validation summary: `artifacts/gentoo-builds/fast-validate/<ts>/summary.json`
- Performance report: `data/gentoo/perf-results/perf_benchmark_results.v1.json`
- Healing summary: `data/gentoo/healing-analysis/summary.json`
- Dashboard: generated by `scripts/gentoo/validation_dashboard.py`

Use the report generator for a single markdown artifact:

```bash
python3 scripts/gentoo/generate-report.py --output docs/gentoo/VALIDATION-REPORT.md
```

## Troubleshooting

## Docker daemon unavailable

- Symptom: `fast-validate` exits with Docker errors.
- Fix: start Docker and re-run.
- Workaround: run `scripts/gentoo/fast-validate.sh --dry-run` to validate pipeline wiring.

## Missing FrankenLibC preload library

- Symptom: hook logs show `FRANKENLIBC_LIB unreadable`.
- Fix: install the ABI library and export `FRANKENLIBC_LIB` to a readable `.so` path.

## Unexpected package skips

- Check:
  - `FRANKENLIBC_PHASE_ALLOWLIST`
  - `FRANKENLIBC_PACKAGE_BLOCKLIST`
  - `configs/gentoo/exclusions.json`

## Structured logs not emitted

- Ensure `FRANKENLIBC_LOG_DIR` and `FRANKENLIBC_PORTAGE_LOG` are writable.
- Hook decisions are written to JSONL in the configured log path.
